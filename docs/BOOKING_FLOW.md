# TimePilot Booking Flow — Client Guide

How clients discover, select, and book appointments through the TimePilot booking interface.

---

## Entry Points

Clients reach the booking page through one of three methods:

| Method | URL Example | Description |
|--------|-------------|-------------|
| **Booking link** | `https://host:3001/?bk=abc123` | Opaque token generated by admin; pre-selects org + user |
| **Direct URL** | `https://host:3001/?org=acme&user=UUID` | Explicit org slug and user ID |
| **Confirmation lookup** | `https://host:3001/?ref=TP-A1B2C3` | View an existing booking by confirmation code |

Optional query parameters:

| Param | Default | Description |
|-------|---------|-------------|
| `duration` | `60` | Appointment length in minutes (minimum 15) |

---

## Booking Flow — Step by Step

### 1. Organization Loading

When the page loads, the SPA resolves the organization:

- **Booking link (`?bk=`):** Calls `GET /api/b/:token` to resolve the token into an org slug and user ID, then loads the organization.
- **Direct URL (`?org=&user=`):** Loads the organization by slug via `GET /api/organizations/slug/:slug`.

The organization's branding (colors, logo, font) is applied to the page.

### 2. Calendar View

A month-view calendar is displayed. The SPA automatically scans up to 12 months ahead to find the first month with availability, so clients land on a useful date immediately.

- **Timezone:** Detected from the client's browser (`Intl.DateTimeFormat().resolvedOptions().timeZone`) and displayed below the calendar (e.g. *"Times shown in America/New_York"*).
- **Navigation:** Month forward/back arrows.
- **Date selection:** Clicking a date fetches available slots.

### 3. Time Slot Selection

Available slots for the selected date appear below the calendar.

- Each slot shows the start time formatted in the client's local timezone.
- Slots are generated server-side based on the user's availability windows minus time blocks and existing appointments.
- Clicking a slot advances to the booking form.

**API call:** `GET /api/organizations/:orgId/slots?userId=...&date=YYYY-MM-DD&timezone=...&duration=...`

### 4. Booking Form

The client fills in their details:

| Field | Required | Notes |
|-------|----------|-------|
| Name | Yes | Display name for the appointment |
| Email | Yes | Receives confirmation and reminder emails |
| Phone | No | For SMS notifications (if configured) |
| Notes | No | Free-text notes for the team member |

A summary of the selected date, time, and duration is displayed above the form.

### 5. Confirmation

On successful submission, the client sees:

- A confirmation card with the reference code (e.g. `TP-A1B2C3`)
- Appointment date and time in their timezone
- The team member they booked with

The browser URL is updated to `?ref=<confirmationRef>` so the page can be bookmarked or shared.

**API call:** `POST /api/organizations/:orgId/appointments` with the selected slot and client details.

**What happens server-side:**
1. The slot is validated (still available — no double-booking)
2. The appointment is created with status `scheduled`
3. A `booking_confirmation` notification is enqueued (fire-and-forget)

If the slot was taken between selection and submission, the client sees a conflict error and can choose another slot.

### 6. Confirmation Lookup

Clients can revisit their booking at any time using the confirmation reference:

```
https://host:3001/?ref=TP-A1B2C3
```

**API call:** `GET /api/appointments/confirm/:ref` — returns client-safe fields only (no internal IDs or admin data).

## Booking Links

Admins generate opaque booking links from the Settings panel. Each link:

- Maps to a specific team member within an organization
- Uses a short random token (no org or user IDs exposed)
- Can include a label (e.g. "Website footer", "Email signature")
- Can be shared as a URL or QR code
- Can be deactivated (revoked) at any time

**QR codes** are generated on demand via `GET /api/b/:token/qr` (inline SVG).

---

## Timezone Handling

- **Detection:** The client detects the browser's timezone automatically.
- **Display:** All times shown to the client are in their local timezone.
- **Storage:** When submitting a booking, the client sends UTC start/end times plus the timezone identifier.
- **Server:** All times are stored in UTC. The timezone is preserved as context for notifications and auditing.
- **Notifications:** Confirmation and reminder emails render times in the appointment's timezone.

## Error States

| Scenario | What the Client Sees |
|----------|---------------------|
| Invalid booking link | "This booking link is invalid or has expired." |
| No availability in 12 months | "No available slots found." |
| Slot taken (double-booking) | Conflict error; client can select another slot |
| Network error | Generic error message with retry option |
| Invalid confirmation ref | "Appointment not found." |

## Admin Panel

Team members access the admin panel at `/admin`. After logging in via OAuth (Google, Apple, or Microsoft), they can:

### Dashboard
- View all organizations they belong to
- Create a new organization
- Switch between organizations

### Settings
- **Organization branding:** Name, description, logo, colors, font, theme presets
- **Profile:** First name, last name, timezone
- **Booking links:** Generate, copy, QR code, revoke
- **Availability:** Create recurring (weekly) or one-time availability windows with buffer time
- **Time blocks:** Create recurring or one-time unavailability periods
- **Email provider:** Enable Gmail or Outlook sending via OAuth consent
- **Email templates:** Customize confirmation, reminder, cancellation, and reschedule emails with live preview

### Appointments
- View upcoming, past, or all appointments
- **Edit:** Update client name, email, phone, notes
- **Reschedule:** Pick a new date and time (validates for conflicts)
- **Cancel:** Cancel with confirmation prompt
- **Create:** Manually add an appointment from the admin side
